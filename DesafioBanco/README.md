Este Desafio, concebido com propósitos didáticos, representa um desafio de usar o paradigma de orientação a objetos para idealizar e modelar um sistema de banco que permita criar tiferentes tipos de contra que façam diferentes tipos de operação, criado em colaboração entre a [Digital Innovation One](https://www.dio.me/) e e o Bradesco, como parte do Bootcamp Bradesco 2025 - Backend com Java.

## Objetivo

Aprender e aplicar os fundamentos de Programação Orientada a Objetos (POO), foram usados os principios de POO da seguinte fomra: 

| Ferramentas | Uso |
| ------------- | ------------- |
| Interfaces | Definir as funcionalides que as contas deveriam ter|
| Herança  |  Tipos de conta herdarem os comportamentos em comum das contas|
| Polimorfismo  | Conta poupança poder render o saldo e a corrente não|
| Encapsulamento  | Não ser possível alterar informações da conta diretamente|
| Exception  | Criar exceptions personalizadas para as operações|
| Enum  | Definir os tipos de conta apenas como Poupança ou Corrente|
